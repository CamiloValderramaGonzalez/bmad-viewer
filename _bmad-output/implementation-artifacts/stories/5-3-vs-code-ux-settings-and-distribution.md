# Story 5.3: VS Code UX, Settings And Distribution

## Outcome

Polish the extension UX and define how it should activate, open and ship in a way that feels native to VS Code.

## Scope

- Decide whether the viewer opens automatically or through commands/settings
- Respect VS Code themes and layout expectations
- Prepare extension packaging, icons, commands and publishing flow

## Current Status

This story is in backlog. The discovery and feasibility discussion is done, but the UX policy and packaging work are still ahead.
