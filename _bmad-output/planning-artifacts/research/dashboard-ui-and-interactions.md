# Dashboard UI And Interactions

## Main UI Layers

- wiki lens for browsing methodology assets
- project lens for metrics, kanban and artifacts
- search modal
- integration modal for external sync flows

## Current Interaction Model

- hash-based routing
- drag-and-drop story movement between BMAD states
- live counters and progress updates
- modal-driven platform connection and sync actions

## UX Tension

The board is now functionally strong, but the UI still needs careful polish to avoid clutter when project metadata and platform integrations grow at the same time.

## Design Direction

The local BMAD board should remain the primary operating surface. Platform integrations should feel secondary, compact and contextual.
